# ubersdr_airsplice

Automated multi-channel audio recorder for [UberSDR](https://ubersdr.org) — connects to a remote UberSDR instance, tunes any number of channels simultaneously, and records demodulated audio to WAV files with a live web UI for playback, management, and monitoring.

---

## How it works

```
UberSDR (remote SDR) --► ubersdr_airsplice (Go) --► WAV recordings
                                 |
                                 └──► web UI  http://<host>:6095
```

- **`ubersdr_airsplice`** — Go service that connects to UberSDR via WebSocket, streams demodulated audio per channel, and writes it to segmented WAV files
- **Smart Record (VOX gate)** — optional SNR-gated recording: only writes audio when signal strength exceeds configurable thresholds, with hysteresis hold timers and pre-roll buffering
- **Scheduler** — optional time-based recording windows per channel (daily, day-of-week, date-range, midnight-spanning)
- **Web UI** — live channel status, SNR meters, FFT spectrum, audio preview, session browser with SNR telemetry timeline, download and delete; served on port 6095

Channels are managed entirely through the web UI and persisted to `channels.json` — no restart required to add or remove a channel.

## Quick start (Docker — recommended)

```bash
curl -fsSL https://raw.githubusercontent.com/madpsy/ubersdr_airsplice/main/install.sh | bash
```

This will:
1. Create `~/ubersdr/airsplice/` and download `docker-compose.yml` and helper scripts
2. Generate a random UI password and inject it into `docker-compose.yml`
3. Create the `recordings/` output directory
4. Pull the latest `madpsy/ubersdr_airsplice` image
5. Start the service

Then edit `~/ubersdr/airsplice/docker-compose.yml` to set your UberSDR URL, and run `./restart.sh`.

Open `http://<host>:6095` to access the web UI.

## Configuration

All configuration is via environment variables in `docker-compose.yml`:

| Variable | Default | Description |
|----------|---------|-------------|
| `UBERSDR_URL` | `ws://ubersdr:8080/ws` | UberSDR WebSocket URL |
| `UBERSDR_PASS` | _(empty)_ | UberSDR bypass password |
| `UI_PASSWORD` | _(empty)_ | Password for write actions in the web UI (auto-generated by `install.sh`) |
| `OUTPUT_DIR` | `/data` | Output directory for recordings inside the container |
| `WEB_PORT` | `6095` | Web UI port |
| `SEGMENT_SECS` | `300` | Rotate to a new WAV file every N seconds; `0` = one continuous file per session |
| `CLEANUP_ALL_DAYS` | `30` | Delete ALL recordings older than N days; `0` = disabled |
| `MAX_STORAGE_MB` | `20480` | Maximum total storage in MB across all channels; `0` = unlimited |

Channels are **not** configured via environment variables — they are added and removed through the web UI and persisted to `channels.json` in the output directory.

## Web UI

Open `http://<host>:6095` in a browser to access:

- **Channel list** — live status for every active channel: frequency, mode, connection state, recording state, SNR meter
- **FFT spectrum** — real-time audio spectrum display per channel
- **Live audio preview** — stream the demodulated audio for any channel directly in the browser
- **Session browser** — recordings grouped by channel and date, with total duration and average SNR; paginated by UTC calendar day
- **SNR telemetry timeline** — per-session SNR/level chart built from the `.jsonl` telemetry log
- **Download** — stream any session or individual segment as a WAV file
- **Delete** — remove individual segments or entire sessions (requires UI password)
- **Retention & quota settings** — configure per-channel or global retention hours and storage quotas from the UI

## Recording modes

### Continuous (default)

Audio is written to disk from the moment the channel connects. A new WAV segment is opened every `SEGMENT_SECS` seconds (default 5 minutes). Set `SEGMENT_SECS=0` for one unbroken file per session.

### Smart Record (VOX-style SNR gate)

When enabled per channel, the recorder monitors the rolling SNR and applies a two-threshold hysteresis gate:

| State | Transition |
|-------|-----------|
| **Idle** | SNR rises above `start_thresh_db` → Arming |
| **Arming** | SNR stays above threshold for `start_hold_sec` → Recording (pre-roll audio included) |
| **Recording** | SNR falls below `stop_thresh_db` → Tail |
| **Tail** | SNR stays below threshold for `stop_hold_sec` → Idle (segment closed) |

An optional `max_record_mins` cap closes the segment and returns to Idle if a single activation runs too long.

### Scheduled recording

When enabled per channel, recording only occurs within configured time windows. Each schedule rule can specify:

- Daily time window (`start_time` … `stop_time`, 24-hour `HH:MM`; midnight-spanning windows supported)
- Day-of-week filter (0 = Sunday … 6 = Saturday; empty = every day)
- Absolute date range (`from_date` … `to_date`, `YYYY-MM-DD`; empty = unbounded)

Multiple rules are combined with OR logic. The timezone is configurable per channel (IANA name, e.g. `Europe/London`; defaults to UTC).

Smart Record and Schedule can be combined: the schedule gate controls *when* the channel is eligible to record, and the VOX gate controls *whether* there is signal worth recording within that window.

## Output files

Each completed recording segment produces three files in the output directory:

| File | Description |
|------|-------------|
| `<uuid>.wav` | PCM audio (mono S16LE, sample rate from UberSDR) |
| `<uuid>.json` | Segment metadata (label, frequency, mode, timestamps, duration, SNR stats) |
| `<uuid>.jsonl` | Telemetry log: one JSON object per line, appended every ~10 s while recording — `{"t":"<ISO>","snr":{...},"level_dbfs":<f32>}` |

A session-level sidecar (`session_<uuid>.json`) is also maintained, grouping all segments that belong to the same continuous recording session.

Orphaned WAV files (e.g. from a crash before the segment was closed) are automatically recovered on startup by reading the WAV header.

---

## Storage management

### Age-based cleanup

Recordings older than the configured retention period are deleted automatically every 5 minutes. Retention can be set globally (default hours) or per channel, and is configurable live from the web UI without restart.

### Quota-based cleanup

When total disk usage exceeds the configured limit, the oldest segments are deleted first until usage is within the limit. Quotas can be set globally (`MAX_STORAGE_MB`) or per channel, and are also configurable from the web UI.

## Building from source

### Docker image

```bash
./docker.sh build          # build madpsy/ubersdr_airsplice:latest
./docker.sh push           # build and push to Docker Hub
./docker.sh run            # run locally (uses env vars)
```

Override the image name:
```bash
IMAGE=myrepo/ubersdr_airsplice:dev ./docker.sh build
```

### Go binary

Requires Go 1.21+

```bash
go build -o ubersdr_airsplice ./...
./ubersdr_airsplice \
    -url ws://sdr.example.com/ws \
    -listen :6095 \
    -output ./recordings
```

All flags:

| Flag | Default | Description |
|------|---------|-------------|
| `-url` | _(required)_ | UberSDR WebSocket URL |
| `-password` | _(empty)_ | UberSDR bypass password |
| `-listen` | `:6095` | HTTP listen address |
| `-output` | `./recordings` | Directory to save recordings |
| `-ui-password` | _(empty)_ | Password for write actions in the web UI |
| `-segment-secs` | `300` | WAV segment length in seconds; `0` = continuous |
| `-cleanup-all-days` | `30` | Delete recordings older than N days; `0` = disabled |
| `-max-storage-mb` | `20480` | Maximum total storage in MB; `0` = unlimited |
